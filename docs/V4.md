# Byteling v4 — "Memory"

v1 gave it a body, v2 a voice, v3 movement. v4 gives it a **past**.

The keystone is a versioned save file (`src-tauri/src/save/mod.rs`): one JSON
document holding a stable pet id, the hatch date, the user's profile, chat
history and its digests, an inbox, care events, and daily mood rollups. It
carries a schema version from day one so future migrations have something to
negotiate with, and a corrupt file is moved aside rather than deleted.

## Shipped

- **A save file.** Everything durable now persists and survives restarts — and
  a Windows reinstall.
- **A user profile.** Tell it your name and a few facts in Settings, and it
  stops calling you "the user."
- **A chat panel.** Right-click the pet → Talk. A real two-way conversation,
  framed as a creature talking back, not an assistant. Older messages fold into
  timestamped summaries so history never grows without bound.
- **Wearables.** Halos, lightsabers, angel wings, a tail — four slots, one item
  each, layered on the base creature and moving with it. A wardrobe lives in the
  clinic.
- **A website.** A landing and download page with the real living creature on
  it, reacting to what you read.
- **A calmer voice.** The pet no longer talks as though you're mid-conversation
  when you've said nothing — it mutters to itself and you overhear.
- **Fixes.** Dizziness no longer trips on a single app switch; idle detection is
  monotonic.

## Deferred to v5

Real age and the birth certificate, an inbox UI, evolution, the weekly letter,
a signed installer, and the widgets platform. The save file's schema already
holds room for most of them.
